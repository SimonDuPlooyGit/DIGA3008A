    • My initial hand drawn wireframes for the home page were inspired by some technical artist portfolios I found mainly from: John O'Really, Arstation and Wolf Van Veen. John O'Really's website in combination with Steam profile pages inspired me to use the idea of a background image behing the content pages with it poking out on the sides. A consideration to the readibility of the page would be to make sure that the background image is not too busy that is distracts from the content page. Since tech art is a very visual thing all of these sites display their work front and centre which I think is the move to make. I additionally decided to add info about me as a top banner before the porfolio work. This is the home page because it is the easiest way to show off what you want for potential employers, which is my target audience. Putting all of the content in on long scrollable page on the homescreen mimics scrolling shorts/reels on social media so since I am showcasing GIFs and videos this composition lends itself to this.

    The idea is for the navigation bar to be stationary/sticky at the top of the page so the user never needs to scroll back up to the top of the page to get to other parts of the site. The other sections of the site can be accessed at the navbar with buttons so that a user can open them in a different tab if they wanted to. To indicate that a user is on a page, the idea is for the current button to be highlighted and to act like a sticky note that you would find in a book or file.

    Next are some wireframes. A wireframe provides a visual plan/idea about how you want to structure your layout. You will have a nice blocked out reference to work from when you start building the website.

    When you land on the home page all of the other content is at most 2 clicks away. The blogs page lists all of the blogs in chronological order with links to them. The same goes for the essays page. The home page and webdesign page has their content on the main page.

    The next step would be to do some stylesheets. After this initial structure has been decided and implemented with HTML the you need to move onto the use of CSS.
